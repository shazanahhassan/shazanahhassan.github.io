Dragon Wealth CSS Index and Style Guide

Hi everyone, and welcome to the new Dragon Wealth CSS Index and Style Guide! This guide is meant to help everyone construct lovely HTML for use in our applications and also as something to refer to when you need a particular look for something.

Here are the basic ground rules and standards for HTML that will have to be adhered to from now on during templating

Rules

  • NO INLINE CSS
  • We will use the <section> tag to seperate out chunks of information
  • Every seperate page will have relevant classes in the <body> tag
  • Let's stay away from tables unless we absolutely have to use them
  • Please use tabs to nest your tags nicely
  • Please be descriptive in your div naming - no stray divs without any classes
  • We should be mobile first so please use the Materialize grids
    •

    Rules for <section>

  • NO INLINE CSS
  • Every <section> should have one class as a unique description of its content
  • Every <section> should have a container class in it
  • Other classes should be reusable classes such as:
  • A component class (e.g. card)
  • A class for width or columns (e.g. s4)
  • A class for alignment override (e.g. left-align)
  • A class for background color/text color overrides (e.g. dw-red)
    •
  • The main heading should be <h3> and subsequent headings <h4> and <h5>
    •
